# AI时代的开发岗

**核心逻辑：** 认为 AI 是生产力倍增器。原来需要 5 个人做的事，现在要求 1 个人带 5 个 AI Agent 做完

**对人的要求：** 不再需要「螺丝钉」（只写前端或只写后端），而是需要「全栈指挥官」。
你必须懂业务、懂架构，然后指挥 AI 去填肉

**残酷现实：** 门槛极具拔高。如果你的产出不能达到以前的 5-10 倍，在老板眼里你就是不合格的



==作为本科生，最大的风险在于：**你还没有建立起核心竞争力，但入门级的「练手」工作已经被 AI 取代了。**==




# 警惕沦为「代码搬运工」的虚假繁荣

你用 Cursor 或 Copilot 生成代码很爽，写作业、做毕设非常快，deepseek可以解决你的代码作业，豆包可以解决你的大部分专业知识。

但是一旦 AI 生成的代码有深层 Bug，或者遇到复杂的系统设计，你将束手无策。


#### 苦练「内功」，做 AI 的「审核员」

在平时的训练中有意的让AI生成有残缺的，错误的代码，尝试自我审核。

## 信息连接VS流量优先

我们不探讨对错，只探讨哪一个更适合我们
1. 谷歌奉行`极简主义`
2. 谷歌的目标是索引全球信息，它更倾向于把你带到**全网最权威**的源头去。你搜索一个技术问题，第一条通常是官方文档或 Stack Overflow
3. 谷歌虽然也靠广告赚钱，但它对广告（Ads）和自然搜索结果的区分非常明显，且广告数量受到严格限制。如果广告的相关性太差，即使出价高也未必能排在第一。
4. 谷歌在自然语言处理（NLP）上处于全球领先地位（如 BERT、Gemini 算法）。它能理解你搜索时的“意图”，即使你输入一串逻辑复杂的句子，它也能抓取到核心点.
5. 谷歌依然是以网页为中心的。只要你的网站内容好，它就能爬取到。

百度：
1. 倾向于“内部优先”。你会发现搜索结果的前几页充斥着百家号、百度百科、百度知道、百度贴吧等自家产品。它更像是一个基于自有内容平台的聚合器，对于站外优质内容的抓取和权重分配逐年下降。

2. 百度在搜索结果中插入广告的比例通常更高，且广告样式有时与自然搜索结果非常相似（虽然有“广告”字样但不够醒目）。这导致用户常常需要翻过大半页“推广”才能看到真实信息

3. 随着国内“APP 孤岛化”，微信、小红书、抖音的内容是不对百度开放的。这导致百度能搜到的信息质量在下降，它不得不大量扶持“百家号”来填补内容空白，但也引入了大量洗稿和低质量营销号内容。

## 搜索技巧

### 1. 关键词搜索
![[Pasted image 20260129150642.png]]![[Pasted image 20260129150715.png]]

### 2. 使用英文
中文互联网的一大特点是信息向应用内部收缩，形成 无法被搜索引擎检索到的“深网”，导致中文开放互联网的信息量小于英文开放互联网的信息量。
![[Pasted image 20260129151515.png]]
### 3.高级搜索

1. 排除：使用减号将不需要的关键词排除在外

![[Pasted image 20260129153407.png]]

2. filestype：限制搜索结果的文件类型。如果你想搜寻关于rust开发的pdf文档，可以使用“rust开发 filetype：pdf”

![[Pasted image 20260129194855.png]]

3. 使用引号将关键词括起来，搜索引擎将强制将其中一个视为一个整体进行搜索，而不是将其拆分成多个关键词。

例如在搜索 明日方舟：终末地的时候，由于搜索引擎有时候会开启联想模式，可能自动屏蔽某些部分，如果你想敲掉某个部分如终末地的时候你可以 
 `明日方舟 “终末地”`


## 为什么不推荐经常使用大模型搜索？

学术性的和最新的消息我们并不推荐使用夸克或者 豆包甚至gpt等大模型进行检索。
虽然在理解能力和语言组织总结方面大模型确实高效，但是由于其黑盒属性，并且目前LLM依旧使用的是Transformer架构，这使得它的知识库是静态的。这导致LLM无法及时获取最新的信息。
在其知识库截止日期后的内容：LLM 对 于一个它不知道的问题往往不会回答“不知道”，而是胡编一个答案出来。在学术上，这往往 是不可忍受的，例如 LLM 会编造不存在的论文，因此在学术场景下使用 LLM 来进行资源粗略 搜索、辅助阅读文献乃至资源整理时，务必慎之又慎。

## 经常用到的检索信息以及提问的平台

1. 最权威的地方永远是官方文档。对于某些团队开发的软件，他们都会花费大量的经历在维护和支持上，因此一般都会有一手的信息和文档在其官网。遇到问题你只需要去官网查阅即可
2. Stack Overflow
 知名程序员问答网站，类似知乎和贴吧的专业版，用户可以在上面提问和快速找到相关问题。
 ![[Pasted image 20260129201636.png]]
 3. github
 github的issue栏目，除了在issue查看大家提出的问题，你也可以fork一个，然后自己完善更多的内容，然后pull request
 ![[Pasted image 20260129202114.png]]
 4. linux.do
 国内知名论坛社区，信息前沿，并且有许多开放的团队在里面发布招聘信息以及公益站点。
 ![[Pasted image 20260129202259.png]]
 5.维基百科
 自由的百科全书，可以替代百度百科
 ![[Pasted image 20260129202440.png]]
 5. OI Wiki，学习算法和数据结构的wiki，请丢掉你手中一节课2个半小时的网课，这里涵盖了大部分的数据结构和算法以及算法本身的知识点，静心的查阅并且多在作题目的时候回来看看，你将有不少收获
